# AGENTS.md — Platypus Service

## Introduction

### What Is This Repo

This repository is a service of the **Platypus platform**. Platypus is a self-service infrastructure provisioning platform: users define desired infrastructure as a declarative config; the platform provisions and operates it. Each service is built by means of a framework.

### Terminology

- **Framework:** A set of files installed in every service repository that defines how services are developed, deployed, operated, and exposed to users — see [Framework Files](#framework-files)
- **Service:** A GitHub repository that provisions and manages a related set of infrastructure resources via a declarative YAML config and Terraform
- **Platform:** The entirety of all services built with the framework

### Your Task

Your task is to implement the service together with the developer. You each bring complementary knowledge: you know the Platypus framework — what files to create, how to structure them, and how everything fits together; the developer knows the service — what infrastructure it should provision and what it should look like to users. Work interactively: the developer describes what they want; you implement it within the framework.

## Framework Files

These files are provided by the framework and are identical across every service — they contain no service-specific logic. Never edit them.

- **`AGENTS.md`:** this file
- **`.github/workflows/run.yml`:** the GitHub Actions workflow that operates the service — see [GitHub Actions Workflow](#github-actions-workflow)

## Service Files

These files are added during development of the service and define what the service does. The files in [Manual Creation](#manual-creation) are created by you or the developer; the files in [Automatic Creation](#automatic-creation) are generated automatically by the framework.

### Manual Creation

The following files must be created by you or the developer — they are not provided by the framework or generated automatically. Ensure each one is present and correct. File names and locations must be exactly as specified.

| File | Name | Directory | Details |
|---|---|---|---|
| Config | `config.yaml` | `/` (repo root) | [Config](#config) |
| Config Schema | `config.schema.json` | `/` (repo root) | [Config Schema](#config-schema) |
| Terraform files | One or more `*.tf` files | `/` (repo root) | [Terraform](#terraform) |

### Automatic Creation

The following files are created automatically by the framework — do not create or edit them manually:

- **`README.md`:** user-facing documentation of the service, generated by the [GitHub Actions Workflow](#github-actions-workflow) from the Config Schema

## Config

> The Config is the developer's domain — they either provide it directly or describe clearly what it should look like. It is the starting point for creating the Config Schema.

## Config Schema

> Agent generates the full Config Schema (structure + all annotations) interactively with the developer. All fields must have `description`, `examples`, and `default`. Schema errors (invalid JSON, invalid JSON Schema structure) are caught by CI on next push.
